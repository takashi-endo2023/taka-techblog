## 読んで変わったこと

### 「なぜPUTはべき等か」を説明できるようになった

PUTはべき等（同じリクエストを何度送っても結果が同じ）だからリトライが安全、という根拠を持って設計できるようになった。

### URIにCRUDを入れる設計がなぜ間違いかわかった

```
# ❌ RPCスタイル
POST /getUser
POST /deleteUser

# ✅ REST
GET    /users/123
DELETE /users/123
```

HTTPメソッドが操作を表現するのがRESTの統一インターフェース。URIは名詞（リソース）だけを表す。

### ステートレス設計の意義がわかった

セッションをサーバに持つと水平スケールが難しくなる。JWTトークンをクライアントに持たせてサーバをステートレスに保つ設計を選ぶようになった。
