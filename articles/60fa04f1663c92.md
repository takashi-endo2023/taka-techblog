正規表現は「なんとなく怖い」と敬遠されがちだが、フォームバリデーション・ログ解析・テキスト変換など実務で使う場面は多い。よく使うパターンをまとめる。

## 基本のメソッド

```ts
// test — マッチするか確認
/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test('user@example.com'); // true

// match — マッチした文字列を取得（gフラグで全件）
'090-1234-5678 と 03-1234-5678'.match(/\d{2,4}-\d{2,4}-\d{4}/g);
// ['090-1234-5678', '03-1234-5678']

// replace — 置換（コールバックで動的置換も可）
'Hello, {name}!'.replace(/\{(\w+)\}/g, (_, key) => values[key] ?? '');

// split — 複数区切り文字で分割
'apple, banana;cherry'.split(/[,;\s]+/);
// ['apple', 'banana', 'cherry']
```

## よく使うバリデーションパターン

```ts
const validators = {
  email:       (v: string) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v),
  phone:       (v: string) => /^0\d{1,4}-?\d{1,4}-?\d{4}$/.test(v),
  postalCode:  (v: string) => /^\d{3}-?\d{4}$/.test(v),
  url:         (v: string) => /^https?:\/\/.+/.test(v),
  password:    (v: string) => /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/.test(v),
  alphanumeric:(v: string) => /^[a-zA-Z0-9]+$/.test(v),
};
```

## キャプチャグループ

```ts
// 日付文字列を分解する
const match = '2026-07-22'.match(/^(\d{4})-(\d{2})-(\d{2})$/);
if (match) {
  const [, year, month, day] = match;
}

// 名前付きキャプチャグループ（ES2018+）
const { groups } = '2026-07-22'.match(/^(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})$/) ?? {};
```

## TypeScriptで型安全なバリデーション

```ts
type FieldName = 'email' | 'phone' | 'postalCode';

function validateField(field: FieldName, value: string): string | null {
  const rules: Record<FieldName, { pattern: RegExp; message: string }> = {
    email:      { pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/, message: 'メールアドレスが無効です' },
    phone:      { pattern: /^0\d{1,4}-?\d{1,4}-?\d{4}$/, message: '電話番号が無効です' },
    postalCode: { pattern: /^\d{3}-?\d{4}$/,              message: '郵便番号が無効です' },
  };
  const rule = rules[field];
  return rule.pattern.test(value) ? null : rule.message;
}
```
