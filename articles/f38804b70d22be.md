JavaScriptで連想配列や集合を扱うとき、オブジェクトやArrayで代用していないだろうか。`Map`と`Set`を使うべき場面を実務目線で整理する。

## Mapの基本

```ts
const map = new Map<string, number>();
map.set('active', 10);
map.set('inactive', 5);

map.get('active'); // 10
map.has('inactive'); // true
map.size; // 2

for (const [key, value] of map) {
  console.log(`${key}: ${value}`);
}
```

オブジェクト（`{}`）との最大の違いは **キーに任意の型を使える** ことと **挿入順が保証される** ことだ。

## Setの基本

```ts
const tags = new Set<string>();
tags.add('TypeScript');
tags.add('React');
tags.add('TypeScript'); // 重複は無視される

tags.size; // 2
tags.has('React'); // true
```

## ArrayとMap・Setの使い分け

### 重複除去

```ts
const arr = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(arr)]; // [1, 2, 3]
```

### 存在確認の速度

```ts
// ❌ Array.includes は O(n)
items.includes('cherry');

// ✅ Set.has は O(1) — 件数に関わらず高速
const itemSet = new Set(items);
itemSet.has('cherry');
```

### カウント集計

```ts
const words = ['apple', 'banana', 'apple', 'cherry', 'banana', 'apple'];

const countMap = new Map<string, number>();
for (const word of words) {
  countMap.set(word, (countMap.get(word) ?? 0) + 1);
}
// Map { 'apple' => 3, 'banana' => 2, 'cherry' => 1 }
```

### キャッシュ（メモ化）

```ts
const cache = new Map<string, string>();

function fetchWithCache(key: string): string {
  if (cache.has(key)) return cache.get(key)!;
  const result = expensiveOperation(key);
  cache.set(key, result);
  return result;
}
```

## 使い分けの指針

| 用途 | 選択 |
|---|---|
| 順序付きリストの管理 | Array |
| キーから値を引く（辞書・キャッシュ） | Map |
| 重複除去・存在確認 | Set |
| キーが文字列以外 | Map一択 |
