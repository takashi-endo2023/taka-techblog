`fetch`はHTTPエラー（404・500）でも`Promise`を`reject`しないため、`res.ok`チェックを忘れると想定外のバグが起きる。実務でよく使うパターンをまとめる。

## 基本のGET（res.okチェック必須）

```ts
// ❌ HTTPエラーを検知できない
const res = await fetch('/api/users');
const data = await res.json();

// ✅ res.ok を確認する
const res = await fetch('/api/users');
if (!res.ok) throw new Error(`HTTP error: ${res.status}`);
const data = await res.json();
```

## 汎用ラッパー関数

```ts
async function apiFetch<T>(url: string, options?: RequestInit): Promise<T> {
  const res = await fetch(url, {
    headers: { 'Content-Type': 'application/json', ...options?.headers },
    ...options,
  });
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json() as Promise<T>;
}
```

## POST リクエスト

```ts
const user = await apiFetch<User>('/api/users', {
  method: 'POST',
  body: JSON.stringify({ name: 'Taka', email: 'taka@example.com' }),
});
```

## タイムアウト（AbortController）

```ts
async function fetchWithTimeout<T>(url: string, timeoutMs = 10000): Promise<T> {
  const controller = new AbortController();
  const id = setTimeout(() => controller.abort(), timeoutMs);
  try {
    return await apiFetch<T>(url, { signal: controller.signal });
  } finally {
    clearTimeout(id);
  }
}
```

## ReactのuseEffectと組み合わせる

```tsx
useEffect(() => {
  let cancelled = false;

  async function load() {
    const data = await apiFetch<Post>(`/api/posts/${postId}`);
    if (!cancelled) setPost(data);
  }

  load();
  return () => { cancelled = true; }; // アンマウント時の更新を防ぐ
}, [postId]);
```

## Next.js App Router でのfetch

```ts
// デフォルトでキャッシュ
const data = await fetch('/api/data').then(r => r.json());

// キャッシュしない
const live = await fetch('/api/realtime', { cache: 'no-store' }).then(r => r.json());

// 60秒でrevalidate
const news = await fetch('/api/news', { next: { revalidate: 60 } }).then(r => r.json());
```

## まとめ

| パターン | ポイント |
|---|---|
| 基本GET | `res.ok`チェックを忘れない |
| POST | `body: JSON.stringify()`でシリアライズ |
| タイムアウト | `AbortController`で実装 |
| React連携 | `cancelled`フラグでアンマウント後を防ぐ |
