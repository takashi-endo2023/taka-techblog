# post-log.md — 日次の軽量ログ（継続を絶やさない用）

最終更新: 2026-06-06

> **これは「毎日1行足すだけ」の超軽量ログ。** 立ち上げ初期の今は、これだけでいい。
> 目的は2つ：(1) **継続できている事実の可視化**（フォロワー数が動かなくても折れないため）／
> (2) **絡みをサボっていないかの監視**（予約投稿だけ出して「やった気」になる罠を防ぐ）。
>
> ⚠️ GA4・Search Console との数字照合は **今はやらない**。フォロワーが2桁後半〜100に
> 近づき、流入が意味を持ち始めてから `operations.md` の本格ログ運用に昇格させる。
> 今はとにかく「止めない」ことだけを記録する。

---

## 書き方（1日1行・30秒で書く）

```
M/D  種別(中身を一言)  /  反応  /  絡み:◯件
```

- **種別**: 気づき / 告知 / 鉱脈 / 一言 / オフ
- **反応**: いいね・リプの手応え（「薄い」「いいね3」程度でOK・厳密でなくていい）
- **絡み:◯件**: 自分から送ったリプ・いいね・フォローの数（★これが一番大事。ゼロの日を見逃さない）
- オフの日は `M/D オフ` だけでいい（書くこと自体が「意図的に休んだ」記録になる）

> **絡み:◯件を必ず書く理由**: 投稿数だけ記録すると「予約で投稿は出した＝やった気」になり、
> 本体である絡み（フォロワー増のエンジン）が抜ける。絡み欄があれば一目で気づける。

---

## フロア（最悪でもこれだけ／止まってない判定ライン）

> 理想の週を目指すのではなく「ゼロの週を作らない」ための最低ライン。

- **平日フロア**: 予約から1本出ている ＋ いいね数個。これさえ満たせば「止まってない」とみなす。
- **水曜**: フロアすら課さない（完全オフ。やりたくてもストック1本まで・絡みなし）。
- 「今週やること」3つが未達でも、フロアを守れていれば **失速していない＝セーフ**。
  自己嫌悪からの停止こそが最大の敵。フロアは自分を責めないための逃げ場。
