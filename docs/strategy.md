## 北極星

**「医療IT × AI ができる希少なエンジニア」**になり、**社内に残るも・転職するも・独立するも「選べる」状態**を作る。

- **二段構え**：社内昇格を実績で狙いつつ、発信で社外の市場価値も育てる。**一社の成否に賭けない**＝最強の安全網。
- **今は種まき期**。2年で「完成」を狙わない。40歳は"土台が固まり選択肢が見え始める通過点"。実るのは40代前半〜半ば。
- **最大のリスクは能力ではなく継続**（失速・燃え尽き）。完璧より継続。

## X 運用戦略（フォロワー 2 → 100 が唯一KPI）

> 本体は**予約投稿ではなく絡み**（リプ・いいね・フォロー）。予約は継続の保険、絡みが増加エンジン。

- **フォロー3層**：①同じ船（近いフェーズ・最初の数人はここ）②少し先（3〜5年先・お手本）③ハブ（議論の中心・リプ欄が集まる場）
- **絡みは負荷順に**：いいね → 体験ベースの短いリプ → 引用・言及
- **リスト運用**：フォローせず観察（①仲間②お手本③ハブ④情報源）
- **鉱脈ネタ**（同僚が部下に／480万削減／本部長モンスター／30歳転身）は「開いた型」で入口だけ小出し。最強の時間帯（土22時等）に置く
- **検索で仲間を探す**：「テックリード」「未経験 転職」「医療 エンジニア」「Claude Code」等で、同じ船・少し先の人に先に絡む

## 長期キャリア（38 → 45 の現実版）

### 時間軸の地図
```
38〜40歳（2年）  土台構築（種まき）   ← 「40歳まで」の正しい目標
40〜43歳         選択肢の現実化（芽）
43〜45歳         実現（収穫・選べる状態の確立）
45歳〜           確立・融合
```

### 勤務先の文脈（2026-06）
- **グループ構造**：医療法人（源流）→ ビル建設会社（親）→ **今の会社**（子・治験営業を分社化）。ヘルスケアで多角展開
- **経営基盤＝極めて安定**：治験募集（医療法人パートナー）で収益途切れず、財務潤沢、潰れにくい子会社。**腰を据えて種まきできる稀な環境**
- **経営の構想**：グループIT全体をこの会社で統括＋システム部を受託事業化（ヘルスケアはIT遅れ＝ブルーオーシャン）
- **自分の立場**：本部長から「早く俺を部長から外せ」「鶴の一声で昇格」＝後任システム部長候補の示唆。**ただし確約ではない（口頭の期待）**

### 今期人事考課の進展（重要）
- **本部長が役員化**（役員会設置）→ **システム部長の席が動く**
- **昇格条件が言語化された**：「**会社全体・治験全体を見渡す視点を持て**」「**安心して任せられるなら社長に進言する**」（＝条件付き・確約でない）
- **視座を上げる打ち手**：①治験ビジネスを数字で理解（CRA/CRP・被験者獲得単価）②IT/AIを"事業ROI"で語る（工数%でなく事業の円・判断速度）③部署を越え事業のボトルネックに踏み込む ④報告を「事業貢献」の言葉で
- **浮かれず、視座を実績で満たしにいく**。`lp-architecture-unification`（LP統一→CRA/CRP正確化）を**事業の言葉で再提示**するのが好機

### 転職書類の現在地（選択肢Bの実弾）
> 「いつでも出せる書類」を持つこと自体が交渉力（社内昇格でも武器）。今すぐ出す必要はない。

- **実績棚卸し（職経書・aboutに反映済み）**：内製化で年480万削減／LangChain×NestJSで自社CRMにAI連携／**Bedrockで機密データのクローズドAI処理**／帳票自動出力（ClaudeCode）／**広告効果測定・分析基盤（マーケ×データ×AI）**／未経験2名育成／グループ会社0→1立ち上げ／自社コーポWordPress刷新／Flutter介護アプリ引き継ぎ

## 全体を貫く5原則
1. **資格はツール、実務は武器、発信は資産**（CLF→SAA→AIFのみ。ML/PMPはやらない）
2. **社内自律化 × 社外可視化の両輪**（一社依存しない＝最強の安全網）
3. **完璧より継続**（失速・燃え尽きが最大の敵）
4. **優先順位は 実務 > 発信 > 自己投資**（均等にやると一番ラクな資格勉強に逃げる）
5. **成果は数字＋発信で二重に可視化**（社内は数字、社外はX/ブログ）

## リスク管理（能力より重要）
| リスク | 対策 |
|---|---|
| 失速（Xが5日で止まった実績）| 1日5分でも止めない。完璧を求めない |
| 燃え尽き（激務＋発信＋資格）| 優先順位を守る。水曜オフ・運動習慣 |
| 資格コレクション化 | CLF/SAA/AIFで打ち止め |
| 一社依存 | 発信で社外価値を作り、選択肢を常に持つ |
| **ドキュメント肥大＝planning procrastination** | docは薄く保つ。設計に逃げそうになったら実行（絡み）に戻る |
