## いま採用している作戦（壁打ちで決めたこと）

- 投稿比率を **気づき8：告知2** に振る（告知中心だと初期は届かない）
- 「開いた型」で問いかけの余白を作る（リプを呼ぶ／完結させすぎない）
- 医療IT×AIの現在進行形の実務ネタを混ぜる（他の人が持っていない独自性）
- note用の鉱脈ネタ（同僚が部下に／480万削減／本部長モンスター／30歳転身）の"入口の一言"だけを X に無料で小出し（全文は note に温存・両立する）
- フォロー戦略：3層で分けて絡む
  - ① 同じ船の人（近いフェーズ・絡める距離）＝初期の最初の数人はここから
  - ② 少し先を行く人（3〜5年先・お手本）＝発信の観察対象
  - ③ 領域のハブ（議論の中心）＝リプ欄が同好の集まる場
- 絡み方は負荷順に：段階1 いいね → 段階2 体験ベースの短いリプ → 段階3 引用・言及

## メモ

- 最大の敵は「失速」。3つに絞り、完璧を求めず、止めないことを最優先する。
- 設計・戦略を詰める作業に逃げそうになったら、実行（泥臭い絡み）に引き戻す。
- 開発フロー自動化（試験/マージ/レビューの全体設計 → 工数削減・平準化）は、まさに今しか書けない実務ネタ。「自動化で何が変わったか」「現場の反応」「設計で迷った判断」を気づき投稿の素材にできる。CLF学習中の気づきも同様。
