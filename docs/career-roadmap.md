## 前提と考え方

- **現在地**: 38歳・エンジニア実務3年・テックリード兼係長（転身組としては異例に速い）
- **強み**: 学習・昇進が速い。医療IT×AI実務という希少ポジション
- **制約**: 「年数でしか買えないもの」（技術の深さ・場数・経営経験・第一人者の認知）がある
- **だから**: 2年で「完成」を狙わない。**40歳は"土台が固まり選択肢が見え始める通過点"**。実るのは40代前半〜半ば
- **最大のリスクは能力ではなく継続**（失速・燃え尽き）。完璧より継続

### 時間軸の地図

```
38〜40歳（2年）  土台構築（種まき）        ← 「40歳まで」の正しい目標
40〜43歳        選択肢の現実化（芽が出る）
43〜45歳        実現（収穫・選べる状態の確立）
45歳〜          確立・融合
```

## 【45歳〜】確立・融合

- 医療IT × AI × 経営を回せる希少人材として、**残るも出るも自分で選ぶ**
- AI導入支援・医療IT顧問を本格化（独立 or 副業継続）
- 資格は**補助のみ・余力で**（中小企業診断士／ITストラテジスト＝経営×IT。北極星には必須でない）

## 🔑 全体を貫く5原則

1. **資格はツール、実務は武器、発信は資産**（CLF→SAA→AIFのみ。集めない。ML/PMPはやらない）
2. **社内自律化 × 社外可視化の両輪**（一社依存しない＝最強の安全網。残るも出るも選べる）
3. **完璧より継続**（失速・燃え尽きが最大の敵。2年"続けられるか"が勝負）
4. **優先順位は 実務 > 発信 > 自己投資**（均等にやると一番ラクな資格勉強に逃げる）
5. **成果は数字＋発信で二重に可視化**（社内は数字、社外はX/ブログ）

## ⚠️ リスク管理（能力より重要）

| リスク | 対策 |
|---|---|
| **失速**（Xが5日で止まった実績）| 1日5分でも"止めない"。完璧を求めない |
| **燃え尽き**（激務＋発信＋資格の詰め込み）| 優先順位を守る。無理なペースにしない |
| **資格コレクション化** | CLF/SAA/AIFで打ち止め。実務・発信に時間を回す |
| **一社依存** | 発信で社外価値を作り、選択肢を常に持つ |

> **合言葉**: 「2年で最強になる」ではなく「2年で最強への"土台"を作る。実るのは数年後」。
> 焦らず、止めず、種をまき続ける。
