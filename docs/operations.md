## 全体フロー（週単位）

```
新記事を書く週
  └─ ブログ記事（src/content/blog/）作成
  └─ Zenn記事（articles/）作成
  └─ 既存記事1本の本文中に今書いた記事へのリンクを貼る（SEO内部リンク）
  └─ git push → GitHub Actions 自動デプロイ・Zenn自動公開
  └─ X告知（ブログURL・スレッド形式）
  └─ Search Consoleで記事URLをインデックス申請

記事を書かない週
  └─ 出稿キュー（下記）の次の行を処理
       1. Zennが未公開なら articles/<hash>.md の published: false → true に変更 → git push
       2. X告知（下記フォーマット）
       3. 出稿キューのチェックボックスを [x] にする
  └─ Search Consoleで記事URLをインデックス申請
```

## X 投稿フォーマット

```
【フック：読む理由を1行で】

① 要点1
② 要点2
③ 要点3

記事はこちら → https://www.taka-techblog.com/blog/<slug>

#タグ1個
```

**フックの書き方**
- 「何をしたか」ではなく「何が起きたか・何に気づいたか」で書く
- 悪い例: 「Claude Codeについて書きました」
- 良い例: 「Claude Codeを入れて一番変わったのはレビュー工数じゃなかった」

**ルール**
- ハッシュタグは1〜2個まで
- ZennのURLは貼らない（ブログURLのみ）
- 自己紹介ポストはプロフィールに固定したまま維持

**一言投稿のネタ例**（週1〜2本・記事告知とは別に投稿）

```
「今日のレビューで久々にN+1踏んだ」
「未経験エンジニアに仕様を書かせたら最初は箇条書きすらできなかった」
「テックリードと係長を兼任すると、1日の会議が5本になる日がある」
「Claude Codeに設計相談したら自分が考えてなかった観点を出してきた」
「医療系スタートアップのリリースは製薬会社の承認が必要なので、バグ修正に3ヶ月かかることがある」
```

### 日常ポストストック2025-05追加（パターン別10本）

> 同じ系統を連投しない。気づき→失敗談→問いかけ→数字 と変化をつける。
> 朝投稿なら冒頭に「最近」「ふと思ったが」を足す。
> リプライ呼びたい時は最後を「みんなどう？」に変える。

#### 転身ストーリー型

```
⑨ 30歳でエンジニアに転身したとき、
「もう遅い」と100回は言われた。

10年経って、今はテックリードをやっている。
遅いかどうかは、続けたかどうかで決まる。
```

#### AIツール所感型

```
⑩ Claude Code を社内に広めたら、
本部長が一番ハマってLPを毎日量産し始めた。

便利な道具は「使う人を変える」けど、
「使う人を選ばない」のが落とし穴だった。
```

```
⑪ Claude Code に CLAUDE.md でルール書くの、
最初は面倒だと思ってた。

3週間続けたら、プロジェクト固有のお作法を
毎回説明しなくていいから、
体感の生産性が1.5倍くらいになった。
```

#### 数字・効果型

```
⑫ 議事録を手で書いていた頃、
コアメンバーが会議の中身に集中できない問題があった。

AI議事録に切り替えたら、誰が出ても品質が同じになった。
後輩に任せられる範囲が一気に広がる。
```

#### 気づき型

```
⑬ テックリード就任して最初の90日でやったこと。

コードを書く時間より、
「現状把握」と「人の話を聞く」時間の方が長かった。

技術力より、まず観察。これが正解だった。
```

```
⑭ 新しいことを始めるとき、
「なぜこの設計にしたか」だけは絶対書き残す。

「何をしたか」はコードを読めば分かる。
「なぜそうしたか」が、未来の自分とチームを救う。
```

#### 失敗→気づき型

```
⑮ 朝会の仕切りを後輩に任せたら、
最初の1ヶ月は完全に崩壊した。

でも更に下の後輩が入って「教える側」になった瞬間、
急に回るようになった。

成長は環境設計で起きる。
```

```
⑯ コードレビューを3年間、一人で全部やってきた。

最近、先輩メンバーに段階的に渡し始めた。
バグはゼロ。意外と回ってる。

任せて初めてわかる。任せないと永遠にボトルネックのまま。
```

#### 問いかけ型

```
⑰ テックリードの仕事、
コードを書く時間と組織を動かす時間が
6:4 → 2:8 になってきた。

これって成長か、キャリアの方向転換か、
正直まだ判断つかない。
```

#### 業界所見型

```
⑱ 外部ベンダーから内製化した3年間。

「お金を浮かせる」目的より、
「自分たちでコントロールできる」価値の方が圧倒的に大きかった。

判断速度が変わる。これが本当の効果。
```
