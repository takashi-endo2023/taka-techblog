## 2. 記事の質・深さ ★★★☆☆

**良い点**:
- `.md`の経験系記事（`clinical-trial-system-performance`・`cloudformation-to-cdk`・`incident-branch-conflict-prod`など）は**現場の一次情報**が詰まっており、ブログとしての価値が高い。「治験データが増えるにつれてシステムが遅くなった」のようなタイトルは読者を引きつける
- 500万円コスト削減・OIDC認証導入など、**数字と成果を使って語れている**点は転職・信頼構築に有効
- 記事にnarrativeに個人の声があり、読み物として成立している

**課題**:
- `.mdx`記事（`javascript-array-methods`・`aws-vpc-subnet-design`・`http-methods-status-codes`など）は**入門チュートリアル的な内容**で、個人ブログの文脈では浮いている。「なぜ私がこれを書くのか」という動機が薄く、SEO目的の量産コンテンツに見える
- 短い記事（4〜5KB）が一部あり、内容が薄い印象。`coaching-junior-engineers`(4.8KB)や`typescript-react-nextjs-book-review`(4.5KB)はテーマに対して書き込みが足りない
- 読了時間の計算が文字数ベース（`/500`）のため、日本語テキストでは実際より短く表示される可能性がある
