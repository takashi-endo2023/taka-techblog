パート1: 画像付き投稿の5パターン

X上でエンゲージメントが2〜3倍違うと言われる「画像付き投稿」。
テキストのみより圧倒的に伸びます。

パターンA: コード画像（最強・エンジニア層に刺さる）

Carbon (https://carbon.now.sh) で30秒で作れる。コードスニペットを綺麗に画像化。

  '禁止事項を明示': true,    // やるな > やれ

パターンE: 実環境スクショ + 注釈

Claude Code 画面・GA4ダッシュボード・Zenn analytics などをスクショして、注釈つけて投稿。「リアル感」が出てエンゲージ強い。

注意
プロジェクト名・社内情報は必ずマスキング。Claude Code 画面ならファイル名やプロンプトの一部を消す。

パート2: ブログ記事のスレッド化（週1本ペース）

既存ブログ記事を5〜10ポストの連続スレッドに分解。既存資産の再利用 + 新規読者獲得のダブル効果。

スレッド構成の鉄則

スレッド設計の3層構造
① 冒頭ポスト: フック（誰の話か・何が起きたか）
② 本編ポスト×3〜5: ストーリー展開、各ポストに「次が気になる」を入れる
③ 締めポスト: 教訓・気づき + ブログURL（フル版誘導）

パート4: 投稿前チェックリスト

画像投稿前

  • ☐ プロジェクト名・社内情報がマスキングされているか
  • ☐ 画像サイズが適切か（Xは16:9または1:1推奨）
  • ☐ モバイルで読める文字サイズか
  • ☐ ファイル形式がPNG / JPG（軽量化）
  • ☐ 代替テキスト（alt）を設定したか

スレッド投稿前

  • ☐ 1ポスト目に強いフックがあるか
  • ☐ 各ポストが「続きを読みたい」終わり方か
  • ☐ 各ポストが単独でも完結する内容か
  • ☐ 最終ポストにブログURLがあるか
  • ☐ ハッシュタグ1〜2個に絞ったか
  • ☐ 投稿時間が22時前後 or 朝8時か

パート5: 月次運用カレンダー（推奨）

第1週: ブログ記事告知（テキスト中心） 第2週: スレッド化投稿（既存記事1本） 第3週: 画像投稿（コード or 引用カード） 第4週: アンケート or 議論喚起ポスト → パターンを循環させて飽きさせない
最後に
すべてを完璧にやろうとしない。
まずは 「Carbonで作ったコード画像 + ブログURL」 から始めるのが最小コスト・最大効果。
